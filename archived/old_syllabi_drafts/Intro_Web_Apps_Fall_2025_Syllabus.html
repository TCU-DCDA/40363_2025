Course Overview

Project‑based introduction to web applications emphasizing audience empathy, accessibility, progressive enhancement, responsive and maintainable front‑end practice, light server interaction, and reflective iteration. AI assistants are used as collaborative tools (design critique, debugging suggestions, refactoring) with attribution.

Final Outcome: A portfolio of 3–4 accessible, responsive, creative web apps (HTML, CSS, JS, light Node/Glitch) with documented decisions (accessibility, performance, user focus).

AI Support Guidelines

  • Use for: examples, debugging hints, refactoring suggestions, accessibility text, performance ideas.
  • Avoid for: unreviewed code submission, bypassing readings, misrepresenting authorship.
  • Document: substantial AI help in README or code comments.

Assessment Philosophy

Mastery / completion model: feedback-driven iteration across dimensions (Accessibility, Responsiveness, Maintainability, Performance Awareness, Version Control Hygiene, Reflection). No punitive grading; progress tracked via checklists and milestone reflections.

Policies

  • Late Labs: 48h grace. Afterward accepted for feedback only (may lag).
  • Collaboration: Discuss & pair; submit only code you personally understand.
  • Accessibility: Evolving checklist (revised after Labs 10, 14, 17) must be met retroactively.

Tools Timeline

  • VS Code (Week 5)
  • Git/GitHub Pages (Week 6)
  • DevTools (Network, Coverage) Weeks 7–12
  • Glitch / Replit (Week 10)
  • Lighthouse / axe (Week 10+)
  • AI Assistant (Week 1+ expanding scope)

Reflections

Short written reflections at Labs 10, 14, 18, and Final: articulate user‑first improvements, constraints managed, next iteration priorities.

Final Project Minimums

  • Core Deployed responsive app (GitHub Pages / Glitch hybrid acceptable).
  • Core Accessibility baseline checklist met (contrast, landmarks, alt text, focus order).
  • Core README: purpose, features, accessibility notes, two identified future performance opportunities.
  • Stretch Documented performance snapshot & optimizations (if Perf Sprint attempted).
  • Stretch Advanced a11y enhancements (ARIA live regions, reduced‑motion adaptations, etc.).

Essential Path Recap: Labs 01–12, 14, 15 + Final Project Core Minimums. Other labs = Stretch: attempt only after Core confidence.
