üåà Color Theory Crash Course

Essential Color Concepts

  • Color Harmony: Colors that work well together (complementary, analogous, triadic)
  • Contrast: Difference between light and dark for readability
  • Saturation: Intensity of color (bright vs. muted)
  • Psychology: Emotional associations with different colors

üéØ Professional Portfolio Color Guidelines

  • 2-3 Colors Maximum: Too many colors look unprofessional
  • 60-30-10 Rule: 60% neutral, 30% primary color, 10% accent
  • Industry Awareness: Tech likes blues/grays, Creative allows bold colors, Finance prefers conservative
  • Accessibility First: Ensure sufficient contrast for all users

Set Up CSS Custom Properties (Variables)

Create a professional color system using CSS custom properties for easy management and compatibility with your LAB08 code:

Add this to the top of your CSS file (after any existing :root section):

/* Color System - Choose ONE palette from above */
/* Add this to your existing CSS file, don't replace existing code */
:root {
  /* Tech Professional Theme */
  --primary-color: #3b82f6;
  --secondary-color: #1e40af;
  --accent-color: #93c5fd;
  
  /* Supporting Colors */
  --text-dark: #1f2937;
  --text-light: #6b7280;
  --background-light: #f9fafb;
  --background-white: #ffffff;
  --border-light: #e5e7eb;
  
  /* Gradients */
  --primary-gradient: linear-gradient(135deg, var(--primary-color), var(--secondary-color));
  --accent-gradient: linear-gradient(135deg, var(--accent-color), var(--primary-color));
}

/* Ensure compatibility with LAB08 layout containers */
header,
main,
footer {
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 2rem;
}

üí° Why CSS Custom Properties?

  • Consistency: Change one value, update everywhere
  • Maintainability: Easy to experiment with different colors
  • Professional: Industry standard approach
  • Flexibility: Can create variations and themes

üõ† Accessibility & Color Contrast

‚ö†Ô∏è Accessibility is NOT Optional

Your color choices must be accessible to all users, including those with visual impairments.

‚úÖ Accessibility Checklist

  • Contrast Ratio: Text should have at least 4.5:1 contrast with background
  • Color Independence: Don't rely solely on color to convey information
  • Testing Tools: Use WebAIM Contrast Checker or browser dev tools
  • Alternative Indicators: Use icons, text, or patterns alongside color
/* Accessible color combinations example */
.accessible-text {
  /* Good contrast: white text on dark background */
  background: var(--primary-color);
  color: white; /* High contrast */
}

.accessible-link {
  color: var(--primary-color);
  text-decoration: underline; /* Visual indicator beyond just color */
}

.accessible-button {
  background: var(--primary-color);
  color: white;
  border: 2px solid var(--primary-color);
  /* Focus state for keyboard users */
}

.accessible-button:focus {
  outline: 3px solid var(--accent-color);
  outline-offset: 2px;
}
