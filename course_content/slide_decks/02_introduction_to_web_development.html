Understanding URLs

The Complete Web Address

https://www.tcu.edu/academics/page.html

  • https:// - The protocol (secure communication)
  • www - The subdomain (optional)
  • tcu - The domain name (unique identifier)
  • .edu - The top-level domain (category)
  • /academics/page.html - The specific file or page
Think of it like a postal address or location path:
USA → Texas → Fort Worth → 76129 → TCU → Reed Hall → Room 219
https:// → www. → tcu → .edu → /academics → /page.html

Both are paths! Each part gets more specific to guide you to exactly what you're looking for.

HTTP vs HTTPS

HTTP
vs
HTTPS

HTTP

HyperText Transfer Protocol

  • Basic web communication
  • Data sent in plain text
  • Not secure
  • Can be intercepted
  • No encryption

HTTPS

HTTP Secure

  • Encrypted communication
  • Data is protected
  • Secure connection
  • Cannot be easily intercepted
  • Required for sensitive data

HTTP: Sending a postcard - anyone can read it

HTTPS: Sending a letter in a sealed envelope

Why Start with Frontend?

  • Immediate Visual Results: See your code come to life instantly
  • Creative Expression: Combine technical skills with design thinking
  • User-Focused: Learn to think about audience and accessibility
  • Foundation Skills: HTML, CSS, and JavaScript are essential for all web work
  • Portfolio Building: Create projects you can show and share
  • Understanding Your Tools: Know what browsers and servers do with your code

Remember: Every backend needs a frontend, but you can build amazing things with just frontend skills! And now you understand the full path your websites take to reach users.

Web Development Best Practices

  • Always use HTTPS: Secure communication builds user trust
  • Test across browsers: Your site should work for all users
  • Think mobile-first: Most users browse on phones
  • Plan your URL structure: Clear paths help users and search engines
  • Optimize for speed: Fast-loading sites provide better experiences
  • Consider accessibility: Make your sites usable by everyone

Remember: You're not just writing code - you're creating experiences that travel through browsers, across networks, and from servers to reach real people with real needs.
